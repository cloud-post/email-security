How To Add Spf Records For Your Domain
    And Increase Email Security

    SPF stands for "Sender Policy Framework," and it's a technical specification that's used to help security administrators monitor email traffic. SPF helps administrators identify which email servers are allowed to send emails from an IP address, and helps determine which servers are blocking emails from that IP address. SPF records are created by domain registrars and are necessary for email authentication

    1. Collect the IP addresses

    Before you begin collecting IP addresses used to send emails through your domain, you must be prepared to collect the list of IP addresses that were used to send emails on your behalf. This will likely contain mailing servers or individuals utilizing third-party mailing servers to send email messages for you. This list should cover all of your email accounts.

    You can check some of the following IP addresses to include in the SPF: the IP addresses of your Internet Service Provider (ISP), an in-office mail server, or any web server hosting your domain. If you are using an ESP, contact your ESP to obtain your IP address.

    2. Compile A List Of Your Sending Domains

    It is possible that your company utilizes various domains in the course of business. You may send some of the official domain names as emails while you may not be allowed to send others.

    Create SPF records that include both private and public IP addresses where you can send messages. This is to prevent spammers from accessing the IP addresses that you don't use if they can t access the ones you use regularly.

    Once you have discovered the IP addresses of the domains that will send your newsletters, live up to your definition of the SPF record by creating it.

    3. Create the Record

    You can create an archive in another SPF version based on the current SPF version. In this case, the current SPF version is stored in the archive in its initial version, so it's 1.

    Whenever you've specified the version type, list the IP addresses you want to use as the address of your hosting service for sending emails on your behalf.

    If there are any third parties that are used to facilitate your email campaign, add them to the list as well. 

    Once you have successfully added the field tags and IP addresses, end the record with all or all tags. This tag is terribly important in the SPF record, as it informs you how to communicate with a server that is not listed in the SPF record. If your server sends a message through your DNS domain, the action determined by the relevant tag will be executed on the error coding server.

    When setting up your SPF record, be aware that you can't exceed 255 characters. However, if you must exceed that limit while adding IP addresses, use the include option. Then create an additional TXT or SPF record for additional. Create another block of 255 characters in the new record.

    When you are finished with the SPF record creation, publish it to the DNS.

    4. Publish the SPF Record

    This means you can publish the report to the DNS, Microsoft Hotmail, Gmail, and many other email service providers.

    To use your DNS server to change your page, follow these instructions.

    1. Visit your domain host provider’s website and log in to your domain account.
    2. Use the DNS management menu.
    3. Select the domain containing the records you want to change.
    4. Open the DNS Manager.
    5. Check the text section and create a TXT record.
    6. Write your domain name in the Host field.
    7. Input your SPF record in the TXT Value field.
    8. Specify the Time to Live (TTL). You may leave it at the default or enter 3600. The Time to Live refers to the lifespan of data stores in a network or a computer.
    9. Click “Add Record” or “Save” to publish the record into your DNS.

    5. Test the SPF Record

    When you are through with the record creation, test it. The test result will display a list of all the servers that are authorized to use your domain name to send emails. If there are no of the IP addresses on the list that you authorized for you to send out emails, update your record.

    In conclusion, SPF records can be created using any DNS provider that supports TXT records and DNSSEC. SPF records can be applied to any domain, subdomain, or IP address. Services that support SPF records include Google's cloud DNS, Cloudflare, and DynDNS.
